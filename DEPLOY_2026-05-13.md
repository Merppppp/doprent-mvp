## 1. รัน Migration SQL 2 ตัว ใน Supabase SQL Editor

ไปที่ Supabase Dashboard → SQL Editor → New query

### ตัวที่ 1 — `2026-05-13_add_verified.sql` (ข้ามได้ถ้ารันแล้ว)
```sql
alter table boutiques
  add column if not exists verified boolean not null default false;

create index if not exists idx_boutiques_verified on boutiques (verified);

update boutiques set verified = true
 where kyc_status = 'verified' and verified = false;

update boutiques set verified = true
 where featured = true and verified = false;
```

### ตัวที่ 2 — `2026-05-13_phase_cd.sql` (สำคัญ ต้องรัน)
เปิดไฟล์ `supabase/migrations/2026-05-13_phase_cd.sql` แล้ว copy ทั้งหมดไปวาง

ตัวนี้ทำหลายอย่าง:
- เพิ่ม `profiles_own_insert` policy (แก้ปัญหา save dress ที่ค้างจาก session ก่อน)

## 5. ทดสอบ

### Renter date picker
1. เปิด https://doprent-mvp.vercel.app/browse → คลิกชุดใดก็ได้
2. ในหน้า /dress/[slug] จะมีกล่อง "เลือกวันที่อยากเช่า"
3. เลือกวันรับ + วันคืน → กดปุ่ม "จองวันนี้ผ่าน LINE — N วัน"
4. LINE จะเปิดพร้อมข้อความ pre-filled

### Seller flow (เปิดร้านใหม่)
1. ออกจาก admin → register ใหม่เป็น user ธรรมดา หรือ ใช้ user ที่ยังไม่มีร้าน
2. ไป /sell → กด "เปิดร้านได้เลย"
3. กรอกข้อมูลร้าน → กดสร้างร้าน
4. ระบบพาไป /sell/kyc → ขั้น 1-4 (ใส่ข้อมูล + อัปโหลดเอกสาร)
5. กดส่ง → ไป /sell/dashboard → จะเห็น "ส่ง KYC แล้ว — รอตรวจ"

### Admin flow (อนุมัติ)
1. ล็อกอินด้วย hgcovuf@gmail.com (admin)
2. ไป /admin → จะเห็น KYC pending, boutique pending, dress pending
3. ไป /admin/kyc → กด Approve ที่ KYC ที่เพิ่งส่ง
4. กลับไป /sell/dashboard ของ seller user → จะเห็น "✓ ผ่าน KYC" และร้านเปลี่ยนเป็น live
5. เพิ่มชุด → ไป /admin/dresses → Approve
6. ดูชุดที่ live ใน /browse

### LINE Clicks dashboard
1. ทักร้านผ่าน LINE จากหน้า /dress/[slug] (จากกล่อง date picker)
2. ไป /admin/clicks → ดูตัวเลขขึ้น

## ⚠️ Notes สำคัญ

- Storage bucket `dress-images` ตั้ง public = true. รูปชุดทุกคนเห็นได้ — ตามที่ design ไว้
- Storage bucket `kyc-docs` private. ใช้ signed URL อายุ 7 วัน — ถ้า admin ตรวจช้ากว่านั้นต้อง refresh URL
- ทุกชุดและทุกร้านสมัครใหม่ จะเป็น status='pending' → admin ต้องอนุมัติก่อนจะแสดงในหน้าเว็บ
- ถ้า admin Approve KYC จะอัตโนมัติ verified=true + status='live' บนร้าน (ไม่ต้องทำสองครั้ง)
