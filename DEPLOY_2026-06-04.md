# Deploy Guide — Business Analytics & Revenue Instrumentation (2026-06-04)

เพิ่มการวัดผลเชิงธุรกิจครบชุดตามที่ทีม + Director ขอ:
แหล่งที่มา (channel), จำนวนผู้เข้าชม/วัน, พื้นที่, MAU/recency,
การจองตามประเภทชุด, commission revenue, และ subscription adoption → revenue.

> branch: `feat/business-analytics`

## 1. รัน Migration ใน Supabase SQL Editor (สำคัญ ต้องรันก่อน)

ไปที่ Supabase Dashboard → SQL Editor → New query → เปิดไฟล์
`supabase/migrations/2026-06-04_business_analytics.sql` แล้ว copy ทั้งหมดไปวาง แล้ว Run

migration ตัวนี้ทำ (idempotent — รันซ้ำได้):

- **profiles**: เพิ่ม `signup_source/medium/campaign/referrer/signup_channel`,
  `last_active_at`, `last_province` (สำหรับ attribution + MAU)
- **page_views** (ตารางใหม่): เก็บ pageview ของ visitor ทุกคน (anon + login)
  พร้อม channel / utm / referrer / province / ip_hash — RLS: anon insert ได้, admin อ่านได้
- **line_clicks**: เพิ่ม `channel/utm_source/referrer/province/session_id`
- **bookings**: เพิ่ม `commission_rate` (default 0.10), `commission_amount` (snapshot),
  `channel` (first-touch ของผู้เช่า) + backfill `commission_amount` ของ booking เดิม
  + policy `bookings_admin_select` (admin อ่านทุก booking เพื่อทำรายงาน)
- **seller_subscriptions** (ตารางใหม่): plan / status / amount / billing_cycle /
  period — รองรับ adoption rate → MRR (พร้อมต่อระบบจ่ายเงินภายหลัง)
- **handle_new_user()**: persist signup attribution จาก `raw_user_meta_data`
- **Reporting views** (สำหรับ SQL editor / BI / สไลด์นักลงทุน):
  `v_daily_traffic`, `v_channel_signups`, `v_traffic_by_province`, `v_mau`,
  `v_daily_bookings`, `v_bookings_by_occasion`, `v_booking_funnel`,
  `v_subscription_revenue`, `v_subscription_adoption`
- **admin_metrics_overview(days)**: RPC (security definer + `is_admin()`) ที่หน้า
  `/admin/metrics` เรียกใช้

## 2. ตั้งค่า ENV (optional)

| Var | Default | ใช้ทำอะไร |
|-----|---------|-----------|
| `PLATFORM_COMMISSION_RATE` | `0.10` | อัตรา commission ที่ snapshot ลง booking ใหม่ |
| `IP_HASH_SALT` | `doprent` | salt สำหรับ hash IP (มีอยู่เดิม) |
| `NEXT_PUBLIC_CLARITY_ID` | — | Microsoft Clarity project id (session replay + heatmap). โหลดเฉพาะ production |
| `NEXT_PUBLIC_GA_ID` | — | Google Analytics 4 Measurement ID (`G-XXXXXXX`). โหลดเฉพาะ production |

### ติดตั้ง Microsoft Clarity (ดู user คลิก/เลื่อนหน้าจริง — ฟรีไม่จำกัด)

1. สมัครที่ https://clarity.microsoft.com → New project → ใส่โดเมน `doprent.com`
2. คัดลอก **Project ID** (เช่น `abcd1234ef`)
3. ใส่ env `NEXT_PUBLIC_CLARITY_ID=abcd1234ef` ใน Vercel (Production)
4. Redeploy → Clarity จะเริ่มอัด session replay + heatmap อัตโนมัติ
   (โหลดเฉพาะ `NODE_ENV=production` — dev/preview ไม่ถูก track)

> PDPA: Clarity mask ข้อความ + input ทั้งหมด default (อัดแค่ layout + การคลิก/เลื่อน)
> แนะนำเพิ่ม consent banner ก่อนเปิดใช้จริงกับลูกค้า

### ติดตั้ง Google Analytics 4 (demographics + Google Ads + BigQuery)

1. สร้าง property ที่ https://analytics.google.com → Admin → Create property
2. เพิ่ม **Web data stream** โดเมน `doprent.com` → คัดลอก **Measurement ID** (`G-XXXXXXX`)
3. ใส่ env `NEXT_PUBLIC_GA_ID=G-XXXXXXX` ใน Vercel (Production) → redeploy
4. เปิด **Google Signals** (Admin → Data Settings) เพื่อดู demographics อายุ/เพศ/ความสนใจ
5. (optional) ตั้ง **Key events** เช่น `booking`, `line_click` เพื่อวัด conversion ใน GA4

> โหลดเฉพาะ production · ปิด auto page_view แล้วยิงเองตอนเปลี่ยนหน้า (นับ SPA navigation ครั้งเดียว)

> **Geo (พื้นที่ผู้เข้าชม)**: ใช้ edge geo headers อัตโนมัติเมื่อ deploy บน **Vercel**
> (`x-vercel-ip-city/country-region`) — ไม่ต้องขอ permission location จาก user
> ถ้า host อื่นที่ไม่ส่ง header กลุ่มนี้ คอลัมน์ `province` จะเป็น null (เมตริกอื่นไม่กระทบ)

## 3. วิธีวัดผลแต่ละเมตริก

| สิ่งที่ Director / ทีมอยากได้ | ดูได้ที่ |
|------------------------------|----------|
| แหล่งที่มา user (IG/FB/Web) | `/admin/metrics` → "แหล่งที่มาผู้เข้าชม" หรือ `v_daily_traffic` |
| visitor/วัน-ชม. | `v_daily_traffic` (group by day) — pageview ทุกคน |
| พื้นที่ตามจังหวัด | `v_traffic_by_province` |
| MAU + สมัคร + recency | `v_mau`, `profiles.last_active_at`, `v_channel_signups` |
| Booking/วัน + ประเภทชุด | `v_daily_bookings`, `v_bookings_by_occasion` |
| Subscription adoption → revenue | `v_subscription_adoption`, `v_subscription_revenue` |
| Conversion จอง + commission | `v_booking_funnel`, `v_daily_bookings.commission_revenue` |

## 4. แปะ Share Link ตาม channel (ฝั่งการตลาด)

วาง link พร้อม `utm_source` เพื่อให้ระบบแยก channel ได้ เช่น:

```
https://doprent.com/?utm_source=ig          → instagram
https://doprent.com/?utm_source=fb          → facebook
https://doprent.com/dress/<id>?utm_source=tiktok&utm_campaign=jun-sale
```

ระบบเก็บเป็น **first-touch** (180 วัน) — สมัคร/จองทีหลังก็ยัง attribute กลับ channel แรกได้

## 5. ทดสอบหลัง deploy

1. เปิดเว็บด้วย `?utm_source=ig` → เช็ก cookie `dp_ft` ถูก set
2. เปลี่ยนหน้า → มี row ใหม่ใน `page_views` (channel = instagram)
3. ล็อกอิน → `profiles.last_active_at` อัปเดต, `signup_channel` backfill
4. สร้าง booking → `bookings.channel` + `commission_amount` ถูกบันทึก
5. เข้า `/admin/metrics` (ต้องเป็น admin) → เห็น KPI + ตารางครบ
