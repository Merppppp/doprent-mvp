# Deploy Guide — Booking + Slip + Business Analytics on Postgres (2026-06-04)

งานชุดนี้ทำ 2 อย่างพร้อมกัน บน stack ใหม่ (เลิก Supabase):

1. **Port booking + upload-slip** จาก Supabase → **Prisma + NextAuth + Cloudflare R2**
2. **Business analytics** (channel attribution, visitor/วัน, geo, MAU, commission revenue,
   subscription adoption) บน Prisma + หน้า `/admin/metrics`

> Stack: Next.js 14 · Prisma · PostgreSQL (Docker) + PgBouncer · NextAuth v5 (JWT) ·
> R2 (S3) · nodemailer · **deploy = Docker บน VPS Ubuntu (ไม่ใช่ Vercel แล้ว)**

---

## 1. Database migration (Prisma — ไม่ใช่ Supabase SQL editor แล้ว)

migration ใหม่: `prisma/migrations/20260604180135_add_bookings_analytics/`
(สร้าง `bookings`, `addresses`, `page_views`, `seller_subscriptions`, enums
`BookingStatus/SubPlan/SubStatus/BillingCycle` + คอลัมน์ attribution/commission/promptpay)

บน VPS ตอน deploy:

```bash
# ใน container หรือก่อน start app — ใช้ DIRECT_DATABASE_URL (bypass PgBouncer)
npx prisma migrate deploy
npx prisma generate          # ถ้า build ยังไม่ได้ generate
```

> `migrate deploy` รัน migration ที่ค้างทั้งหมดแบบ idempotent (ไม่สร้าง migration ใหม่)

## 3. สลิปการจอง (payment slip) → R2

- เดิมเก็บใน Supabase Storage (private bucket + signed URL)
- ตอนนี้: `uploadSlip()` server action validate magic-bytes + ≤5MB แล้วอัปขึ้น R2 key
  `slips/{bookingId}/{uuid}.ext` → เก็บ URL ลง `bookings.slip_path`
- **หมายเหตุความเป็นส่วนตัว:** R2 public bucket = ใครมี URL ก็เปิดได้ (key เป็น uuid เดายาก)
  ถ้าต้องการเข้มกว่านี้ ค่อยเปลี่ยนเป็น private bucket + signed URL ทีหลัง

## 4. Geo (จังหวัดผู้เข้าชม) — ตั้งที่ nginx (เพราะไม่ใช่ Vercel แล้ว)

track route อ่าน header ตามลำดับ: `x-geo-region` → `x-geo-city` → `cf-*` (ไม่มี Vercel แล้ว)
ดังนั้นบน VPS ให้ nginx (GeoIP2 module) ใส่ header ให้:

```nginx
# ต้องลง libnginx-mod-http-geoip2 + ฐาน GeoLite2-City.mmdb
geoip2 /etc/nginx/geoip/GeoLite2-City.mmdb {
  $geo_country country iso_code;
  $geo_region  subdivisions 0 names en;
  $geo_city    city names en;
}
location / {
  proxy_set_header X-Geo-Country $geo_country;
  proxy_set_header X-Geo-Region  $geo_region;
  proxy_set_header X-Geo-City    $geo_city;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_pass http://127.0.0.1:3000;
}
```

ถ้าไม่ตั้ง geoip → คอลัมน์ `province` เป็น null (เมตริกอื่นไม่กระทบ)

## 5. Analytics tools (ฟรี, โหลดเฉพาะ production)

- **Microsoft Clarity**: clarity.microsoft.com → Project ID → `NEXT_PUBLIC_CLARITY_ID`
- **GA4**: analytics.google.com → Measurement ID `G-XXXX` → `NEXT_PUBLIC_GA_ID`
  (เปิด Google Signals เพื่อดู demographics)
- แปะ share link ต่อ channel: `?utm_source=ig` / `fb` / `tiktok` → ระบบเก็บ first-touch 180 วัน

## 6. ทดสอบหลัง deploy

1. เปิดเว็บด้วย `?utm_source=ig` → cookie `dp_ft` ถูก set → มี row ใน `page_views` (channel=instagram)
2. ล็อกอิน → `users.last_active_at` อัปเดต, `signup_channel` backfill
3. จองชุด → `bookings.channel` + `commission_amount` ถูกบันทึก
4. อัปสลิป → ไฟล์ขึ้น R2, `bookings.status` = payment_review
5. `/admin/metrics` (admin) → เห็น KPI + ตารางครบ
